# Introduction

habitat is building extensions to AT protocol that brings data ownership to a greater degree than the protocol currently offers. Our server is an extension to standard AT protocol infrastructure and a developer platform that is built with two use-cases in mind:

1. Building local-first apps for end-users in order to provide better privacy guarantees.
2. Building applications for organizations that give the organizations control over their data.

These two use-cases are interrelated, but not the same, and in the long term we hope our platform can evolve to meet the needs of both.

You may be interested in using habitat to build your app if:

- you want to quickly get started building a product experience without worrying about where to store the backing data, implementing permissions, implementing identity and auth
- you want to build applications that are interoperable with the ATmosphere but use private data
- you want to build in community / group / organizational settings
