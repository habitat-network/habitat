# Sync & local-first software

## Local-first software 

The sweet spot for building on habitat is to build social, [local-first](https://www.inkandswitch.com/essay/local-first/#5-the-long-now) applications. Local-first means that the user's device is treated as the primary source, and data is synced back up to their PDS. It is expected that data travels between user PDS'es and the client application, and does not get stored on application servers. We aim to provide the right developer primitives to make this possible, so if you have a use case that is not currently supported, please reach out to us.

## Sync 

⚠️  Under construction ⚠️ 

Real-time collaboration on the web is now a standard. We are developing peer2peer libraries that operate on AT protocol data (records) and make use of CRDTs to easily create real-time syncing applications no top of habitat. Our first demo of this is the [docs](https://habitat.network/habitat/docs/#/login) app and we will be formalizing these APIs shortly.
