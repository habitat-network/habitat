# What is Habitat?

Habitat is building an open-source and self-hostable data ownership layer for organizations. Our mission is to make data ownership part of the internet's infrastructure. Habitat gives communities and organizations a data server with identities, user data repositories, fine-grained permissioning, and syncing mechanisms built-in, and Habitat is made with open protocols and standards. That way, groups can focus on building and using the applications and experiences that serve their needs, without giving up control of their data, now or at any point in the future. Habitat is an adaptation of [AT Protocol](https://atproto.com) built specifically for the organizational and community use case.

### What can I do with the Habitat?

If you are:
- someone creating custom or niche applications for a community you are a part of
- a developer seeking to give your users more customization and value from your product through data APIs
- an AT protocol builder hoping to serve organizations and groups beyond the big-world use cases

then Habitat enables you to pursue these goals by providing the backend primitives and frameworks that underpin communitiy and organizational data and allow real applications to be built on top, without needing to build them from scratch.

### For AT Protocol developers:

At Habitat, we believe that the primitives afforded by AT Protocol--data ownership, data interoperability, credible exit--are just as important to _collectives_ as they are to individuals. Beyond using the internet to connect with each other socially, we also use the internet to work, collaborate, organize towards shared goals and ambitions. While AT protocol provides better affordances at an individual level, for larger groups and groups with governance, there are additional considerations that come into play: data and identities that persist as people come and go, sub-groups and roles that permeate across application boundaries, and complicated permissions, often with inheritance structures. Habitat extends AT protocol, and reconfigures some of its main components to deliver on these goals. See our Architecture docs for more info.
