# AuthZEN Access Evaluation Design

**Goal:** Add an AuthZEN Access Evaluation endpoint to Pear that checks if a DID has access to an AT URI record.

**Architecture:** A new `internal/authzen/` package with its own `Server` struct (following the `clique.NewServer` / `org.NewServer` pattern). The handler accepts the standard AuthZEN Access Evaluation request body, maps AuthZEN entities to Habitat concepts, and delegates to `pear.Pear.HasPermission`. A new lexicon under `lexicons/network/habitat/authzen/` defines the XRPC procedure.

**Tech Stack:** Go, AT Protocol Lexicon + XRPC, gorilla/mux, `api/habitat` generated types

---

## Entity Mapping

| AuthZEN Concept | Habitat Concept | Field |
|---|---|---|
| Subject | DID (user) | `subject.id` = `"did:plc:abc123"` |
| Resource | AT URI (record) | `resource.id` = `"habitat://did:plc:def456/net.example.collection/rkey123"` |
| Action | Permission type | `action.name` = `"can_read"` |
| Decision | bool | `decision` = `true` (allowed) / `false` (denied) |

Only `can_read` is supported initially. Unknown actions return `{decision: false}` (deny-closed).

## Lexicon

One file: `lexicons/network/habitat/authzen/evaluate.json`

Named defs for `#subject`, `#resource`, `#action` entities (same pattern as `permissions/listPermissions.json` uses for `#permission`). The main def is a `procedure` (POST with JSON body).

Generated types (in `api/habitat/authzen_evaluate.go`):
- `NetworkHabitatAuthzenEvaluateInput` тАФ with Subject, Resource, Action, Context fields
- `NetworkHabitatAuthzenEvaluateOutput` тАФ with Decision (bool) and optional Context
- `NetworkHabitatAuthzenEvaluateSubject` тАФ Type, Id, Properties
- `NetworkHabitatAuthzenEvaluateResource` тАФ Type, Id, Properties
- `NetworkHabitatAuthzenEvaluateAction` тАФ Name, Properties

## New Package: `internal/authzen/`

One file: `internal/authzen/server.go`

```go
type Server struct {
    authMethods struct {
        oauth       authn.Method
        serviceAuth authn.Method
    }
    pear pear.Pear
}

func NewServer(...) *Server
func (s *Server) EvaluateAccess(w http.ResponseWriter, r *http.Request)
```

Handler flow:
1. `authn.Validate(w, r, oauth, serviceAuth)` тАФ authenticate caller
2. `json.NewDecoder(r.Body).Decode(&input)` тАФ decode request body into `NetworkHabitatAuthzenEvaluateInput`
3. Parse `input.Subject.Id` as `syntax.ParseDID()` тАФ the requester being checked
4. Parse `input.Resource.Id` as `habitat_syntax.ParseHabitatURI()` тЖТ `ExtractParts()` тЖТ (owner, collection, rkey)
5. Validate `input.Action.Name` тАФ must be `"can_read"`, else return `{decision: false}`
6. Call `s.pear.HasPermission(ctx, caller, requester, owner, collection, rkey)`
7. Return `{decision: result}`

## Route Registration

In `cmd/pear/main.go`:
```go
import "github.com/habitat-network/habitat/internal/authzen"
authzenServer := authzen.NewServer(pear, oauthServer, authn.NewServiceAuthMethod(dir))
mux.HandleFunc("/xrpc/network.habitat.authzen.evaluate", authzenServer.EvaluateAccess)
```

## Non-Goals

- AuthZEN Subject Search, Resource Search, Action Search, or batch Evaluations APIs
- Standard AuthZEN HTTP binding (non-XRPC path)
- Fine-grained authorization on who can call the evaluate endpoint (any authenticated user)
